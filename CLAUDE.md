### Lokales Setup

`.env` vom Server holen und für lokale Entwicklung anpassen:

```bash
scp -i ~/.ssh/server_key root@91.98.192.233:/opt/business-hub/.env .env
```

Danach diese Werte ändern:

```
FLASK_ENV=development
DEBUG=True
PERSIST_BASE=data
LOG_DIR=logs
USE_POSTGRES=false
SENTRY_DSN=
CALENDAR_DRY_RUN=true
```

**`CALENDAR_DRY_RUN=true`** blockiert alle Google Calendar Write-Operationen (create, update, delete) und gibt Dummy-Responses zurück. Reads (Verfügbarkeit, Events) laufen weiter gegen die echte API. **In Production NICHT setzen** (default: `false`).

## Architektur-Regeln (KRITISCH)

### 1. Daten IMMER via Services - NIE direkte JSON-Manipulation

```python
from app.core.extensions import data_persistence

scores = data_persistence.load_scores()
data_persistence.add_coins(username, 50)
```

Wichtigste Services:
- **data_persistence** - Scores, Coins, Badges, User-Stats
- **tracking_system** - Buchungen tracken
- **achievement_system** - Badges/XP vergeben
- **audit_service** - Logging aller Aktionen
- **notification_service** - Benachrichtigungen

Vollständige Service-Liste mit Imports: `docs/DEVELOPER_GUIDE.md`

### 2. Buchungs-Kette (IMMER alle drei nach erfolgreicher Buchung!)

```python
tracking_system.record_booking(booking_data)
achievement_system.check_achievements(username, 'booking', context)
audit_service.log('booking_created', username, details)
```

### 3. Dual-Write (JSON + PostgreSQL)

Migration ~61% fertig. Neue Daten IMMER in beide Systeme schreiben.
Details: `docs/MIGRATION_STATUS.md`

## Häufige Fehler (VERMEIDEN!)

1. **Neue JSON-Datei erstellen** statt bestehenden Service erweitern
2. **Business-Logik in Routes** statt in Services schreiben
3. **`print()` verwenden** statt `app.logger` / `logging.getLogger(__name__)`
4. **Template nicht von Base erben** lassen
5. **Google Calendar ohne Error-Handling** aufrufen (immer try/except + Retry)
6. **Particles.js falsch konfiguriert** - Max 30 Partikel, speed 0.5, hover DEAKTIVIERT
7. **PERSIST_BASE mit /persistent** am Ende setzen - Code fügt das automatisch hinzu
