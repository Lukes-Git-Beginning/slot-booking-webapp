<objective>
Fix inconsistent config access across all Finanzberatung services and fix the DSGVO file deletion path bug.

Purpose: Three bugs exist from v1.0: (1) four services use `current_app.config.get('FINANZ_*')` which always returns None/defaults because Flask only loads `Config`, not `FinanzConfig`; (2) DSGVO `execute_deletion()` builds the wrong file path (extra `/persistent/` segment + missing `session_id`) so files are never actually deleted; (3) `FINANZ_LLM_ENABLED` env var has no effect because services read it through the broken Flask config path.

Output: All finanz services use direct `finanz_config` class imports, path helpers centralized on `FinanzConfig`, DSGVO deletion actually finds and removes files.
</objective>

**Step 1 — Add classmethods to FinanzConfig in `app/config/base.py`:**

Add three classmethods to the `FinanzConfig` class (before the singleton instantiation at bottom of file). `os` is already imported at top of file.

```python
@classmethod
def get_upload_dir(cls, session_id: int) -> str:
    """Return the session upload directory path."""
    return os.path.join(Config.PERSIST_BASE, cls.FINANZ_UPLOAD_DIR, str(session_id))

@classmethod
def get_file_path(cls, session_id: int, filename: str) -> str:
    """Return the full filesystem path for a stored document."""
    return os.path.join(cls.get_upload_dir(session_id), filename)

@classmethod
def get_chromadb_path(cls) -> str:
    """Return the ChromaDB persistence directory path."""
    return os.path.join(Config.PERSIST_BASE, 'chroma_db')
```
