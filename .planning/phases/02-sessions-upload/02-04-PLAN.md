  <action>
1. Create `app/services/finanz_sse_service.py` with `SSEManager` class:
   - Use `logging.getLogger(__name__)` (NEVER print)
   - Initialize with Redis URL from `os.getenv('REDIS_URL', 'redis://localhost:6379/0')`
   - Use Redis DB 0 for pub/sub (same as sessions -- pub/sub is separate from key-value, no conflict)
   - Graceful degradation: if Redis unavailable, log warning and make publish a no-op

   Methods:
   - `__init__(self, redis_url=None)`:
     Connect to Redis with error handling. Set `self.redis = None` if connection fails.
   - `publish(self, session_id, event_type, data)`:
     Channel: `finanz:session:{session_id}`
     Message format: JSON `{event: event_type, data: data, timestamp: iso_now}`
     If redis is None, log warning and return (graceful degradation).
   - `stream(self, session_id)`:
     Generator that yields SSE-formatted events.
     Subscribe to channel `finanz:session:{session_id}`.
     Use `pubsub.get_message(timeout=30)` in a loop (not `pubsub.listen()` which blocks forever).
     On message: yield `f"event: {event_type}\ndata: {json_data}\nid: {timestamp}\n\n"`
     On timeout (no message): yield heartbeat `": heartbeat\n\n"`
     Include `retry: 5000\n` on first event (tells browser to reconnect after 5s)
     Clean up pubsub on GeneratorExit.
   - `format_upload_event(self, document)`:
     Helper to format a FinanzDocument into SSE event data dict:
     `{id, original_filename, file_size, mime_type, status, created_at, session_id, customer_name}`

   Create module-level singleton:
   ```python
   sse_manager = SSEManager()
   ```

2. Create `app/routes/finanzberatung/sse.py` with sub-blueprint `sse_bp = Blueprint('finanz_sse', __name__)`:
   - `@require_login` on all routes (only consultants see SSE stream)

   Routes:
   - `GET /sse/stream/<int:session_id>` -- SSE streaming endpoint
     - Verify user has access to this session (own session or admin)
     - Return `Response(stream_with_context(sse_manager.stream(session_id)))` with headers:
       - `Content-Type: text/event-stream`
       - `Cache-Control: no-cache`
       - `X-Accel-Buffering: no` (disables nginx buffering per RESEARCH.md pitfall 3)
       - `Connection: keep-alive`

   - `GET /sse/global` -- Global SSE stream for toast notifications
     - Subscribe to all sessions owned by current user (use pattern subscribe `finanz:session:*` and filter by ownership in the generator)
     - Alternative simpler approach: subscribe to user-specific channel `finanz:user:{username}` and have publish() also publish to the user channel
     - Choose the user-channel approach: cleaner, less Redis overhead

   Update `SSEManager.publish()` to also publish to `finanz:user:{opener_username}` channel (fetch opener from session). This enables global toast notifications on any page.

3. Modify `app/routes/finanzberatung/upload.py` -- Add SSE publish after successful upload:
