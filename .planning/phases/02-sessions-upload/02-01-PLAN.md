   - Use `logging.getLogger(__name__)` (NEVER print)
   - Use `from app.models import get_db_session` for all DB access
   - Import models: `FinanzSession, FinanzUploadToken, SessionStatus, TokenType`
   - Import config: `from app.config.base import FinanzConfig as finanz_config`

   Methods:
   - `create_session(opener_username, customer_name, session_type, appointment_date) -> FinanzSession`
     Creates session with opener auto-set, status=ACTIVE. Returns committed session.
   - `get_session(session_id, username=None) -> FinanzSession | None`
     Gets session by ID. If username provided, filters by opener_username (own sessions only per CONTEXT.md). Admins bypass this filter (pass username=None).
   - `list_sessions(username=None, status=None) -> list[FinanzSession]`
     Lists sessions. If username provided, filter by opener_username. Optional status filter. Order by appointment_date DESC.
   - `update_notes(session_id, field, content, username) -> bool`
     Updates t1_notes or t2_notes. Validates: only opener can edit t1_notes, only closer can edit t2_notes. Returns success bool.
   - `transition_status(session_id, new_status, username) -> FinanzSession`
     Calls session.transition_to(). Wraps in try/except with rollback. Returns updated session.
   - `assign_closer(session_id, closer_username) -> FinanzSession`
     Sets closer_username on session.
   - `get_session_documents(session_id) -> list[FinanzDocument]`
     Returns documents for a session, ordered by created_at DESC.

   All methods: try/except with db.rollback() on error, db.close() in finally, logger.error on failure.

2. Create `app/services/finanz_upload_service.py` with class `FinanzUploadService`:
   - Use `logging.getLogger(__name__)` (NEVER print)
   - Import: `secrets, uuid, hashlib, os, io, base64, qrcode, magic` (python-magic)
   - Import models: `FinanzUploadToken, FinanzDocument, TokenType, DocumentStatus`
   - Import config: `from app.config.base import FinanzConfig as finanz_config`

   Methods:
   - `generate_token(session_id, token_type, db_session=None) -> tuple[FinanzUploadToken, str]`
     Generates `secrets.token_urlsafe(48)`, creates FinanzUploadToken with TTL from FinanzConfig (T1/FOLLOWUP/T2 map), max_uploads from config.
     Generates QR code using same pattern as `security_service.py:144-157` (qrcode library, base64 PNG).
     Upload URL: `{FINANZ_UPLOAD_BASE_URL}/finanzberatung/upload/{token}` where base URL comes from env var `FINANZ_UPLOAD_BASE_URL` defaulting to `https://berater.zfa.gmbh`.
     Returns (token_object, qr_base64_string).
   - `validate_token(token_value) -> tuple[bool, FinanzUploadToken | None, str]`
     Looks up token by value. Returns (is_valid, token_or_none, error_message).
     Error messages in German: 'Token nicht gefunden', 'Token abgelaufen', 'Upload-Limit erreicht', 'Token deaktiviert'.
   - `validate_file(file_storage) -> tuple[bool, str, str]`
     Validates file via magic bytes using `python-magic` (NOT extension-based).
     Reads first 2048 bytes, calls `magic.from_buffer(header, mime=True)`.
     Allowed MIME types: `application/pdf, image/jpeg, image/png, image/tiff, image/heif, image/heic`.
     Also checks file size against `finanz_config.FINANZ_MAX_FILE_SIZE_MB * 1024 * 1024`.
     Returns (is_valid, detected_mime, error_message). Error messages in German.
   - `store_file(file_storage, session_id, token_id, original_filename, mime_type) -> FinanzDocument`
     Generates UUID filename: `{uuid4()}.{extension_from_mime}`.
     Computes SHA-256 hash of file content.
     Checks for duplicate hash in same session (dedup).
     Saves file to `{PERSIST_BASE}/{FINANZ_UPLOAD_DIR}/{session_id}/{uuid_filename}`.
     Creates FinanzDocument record with all fields populated, status=UPLOADED.
     Increments token.upload_count.
     Returns the FinanzDocument.
   - `get_active_token(session_id) -> FinanzUploadToken | None`
     Returns the currently active (valid) token for a session, or None.
   - `can_generate_followup(session_id) -> bool`
     Per CONTEXT.md: followup token only available after T1 token expires or is fully used. Checks if any active T1 token exists -- if yes, returns False.
   - `deactivate_token(token_id) -> bool`
     Sets is_active=False on token.

   File storage must create directories recursively (`os.makedirs(..., exist_ok=True)`).
   All methods: proper error handling with logging.
