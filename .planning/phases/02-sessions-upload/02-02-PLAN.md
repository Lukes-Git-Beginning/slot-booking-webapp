  <action>
Create `app/routes/finanzberatung/sessions.py` with a sub-blueprint `sessions_bp = Blueprint('finanz_sessions', __name__)`.

All routes require `@require_login` from `app/utils/decorators.py`. Use `session['username']` for current user (this is the Flask session, not FinanzSession).

Routes:

1. `GET /sessions` -- Session list page
   - Call `session_service.list_sessions(username=current_user)` (own sessions only)
   - If `is_admin`, call with `username=None` (see all)
   - Render `finanzberatung/session_list.html` with sessions list

2. `GET /sessions/create` -- Session creation form
   - Render `finanzberatung/session_create.html`

3. `POST /sessions/create` -- Handle session creation
   - Extract: customer_name (required, strip whitespace), session_type (from dropdown), appointment_date (parse from input)
   - Validate: customer_name not empty, session_type in ['erstberatung', 'folgeberatung']
   - Call `session_service.create_session(opener_username=current_user, ...)`
   - Auto-generate T1 upload token: `upload_service.generate_token(session.id, 't1')`
   - Flash success message in German, redirect to session detail

4. `GET /sessions/<int:session_id>` -- Session detail page
   - Call `session_service.get_session(session_id, username=current_user)` (own sessions check)
   - If admin: `get_session(session_id, username=None)`
   - 404 if not found
   - Get active token: `upload_service.get_active_token(session_id)`
   - Get documents: `session_service.get_session_documents(session_id)`
   - Check followup availability: `upload_service.can_generate_followup(session_id)`
   - Render `finanzberatung/session_detail.html` with session, token, qr_code, documents, can_followup

5. `POST /sessions/<int:session_id>/generate-token` -- Generate new token (AJAX)
   - Extract token_type from request JSON ('t1', 'followup', 't2')
   - For followup: verify `can_generate_followup()` returns True, else 400
   - Call `upload_service.generate_token(session_id, token_type)`
   - Return JSON: `{token_id, token_value, qr_base64, expires_at, token_type}`

6. `POST /sessions/<int:session_id>/notes` -- Auto-save notes (AJAX)
   - Extract `field` ('t1_notes' or 't2_notes') and `content` from request JSON
   - Call `session_service.update_notes(session_id, field, content, username=current_user)`
   - Return JSON: `{success: true/false, message: '...'}`

7. `POST /sessions/<int:session_id>/status` -- Transition status (AJAX)
   - Extract `new_status` from request JSON
   - Call `session_service.transition_status(session_id, new_status, username=current_user)`
   - Return JSON: `{success: true, new_status: '...'}`

8. `POST /sessions/<int:session_id>/assign-closer` -- Assign closer
   - Extract `closer_username` from form
   - Call `session_service.assign_closer(session_id, closer_username)`
   - Flash success, redirect to session detail

9. `GET /sessions/<int:session_id>/documents` -- Get documents as JSON (for SSE reconnect)
   - Optional query param `since` (ISO timestamp) to filter by created_at
   - Returns JSON array of document objects

Instantiate services at module level:
```python
session_service = FinanzSessionService()
upload_service = FinanzUploadService()
```

Error handling: All routes wrap service calls in try/except, flash German error messages, redirect or return JSON error with appropriate status code. Use `logging.getLogger(__name__)`.
